FOOTER

I am really glad that this was a pairing challenge because it allowed my pair and I to switch off between each requirement seamlessly. I think one of the trickiest parts of this assignment was deciding how to facilitate changes made to the mobile version of the app using a media query. Adding the media query to our CSS was pretty simple, but deciding what changes need to be inside the media query was trial and error at first.

In our first iteration of CSS we added our "min-width: 100%;" code to the nav bar container selector (#side-a) outside of our media query. This kept the container at same size even when resizing to a screen size of below 480px (or max-width conditional size in the media query). Since the container stayed at its original 100% size even while resizing, there was no room for an inline list display. Furthermore, this caused the text to be cut off by the pictures and text in the 'content' container. Fixing this was tricky, but once we found the solution, it was definitely an 'aha' moment for me. The min-width condition needed to be placed in the media query, so that when the browser is resized it occupies 100% of the screen width while displaying the list inline.

Overall this challenge helped me understand the visual transition from a larger screen to a mobile screen. I have a new found respect for fluidity in websites, especially ones that retain shape in all points of resizing. You will notice that on the way to our mobile size, the website doesn't retain its shape as well as it should. I guess this would be the next step in improving the design.
